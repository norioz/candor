Video Call 1 to N with WebRTC

This web application consists of an one to many video call using WebRTC. This is an implementation of a video broadcasting application. The application has two types of users: a single peer that sends media (the Presenter) and N peers receiving the media of the Presenter (the Viewers). 1 WebRtcEndpoint is set in SendOnly mode, for the Presenter, and its stream is sent to N WebRtcEndpoints, for the N Viewers. Thus, the media pipeline is composed of N+1 interconnected WebRtcEndpoints.

  1. Open this page with a WebRTC compliant browser.
  2. If you would like to be the Presenter, click the Presenter button. Because there can only be one Presenter, if a Presenter is alerady present, an error is signalled to the console.
  3. Grant the access to the camera and mic. After the SDP negotiation, the feed should start.
  4. In a different browser instance, click the Viewer button to see the Presenter's stream.
  5. Click on the Stop button to end the feed. When the Presenter clicks on Stop, all Viewers end communication as well.
